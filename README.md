# E-commerce  — электронная торговля, или электронная коммерция. Ядро для интернет-магазина.

## Описание:

Приложение для создания ядра интернет-магазина.

## Возможности:

- ****Товары (Product)****
Класс хранит характеристики товара (название, описание, цена, количество)
- Абстрактный класс BaseProduct является родительским классом для класса Product
- **«Смартфон» (Smartphone)** - 
дочерний класс Product, который хранит характеристики товара (название, описание, цена, количество, производительность, модель, объем встроенной памяти, цвет)
- **«Трава газонная»** **(LawnGrass)** - 
дочерний класс Product, который хранит характеристики товара (название, описание, цена, количество, страна-производитель, срок прорастания, цвет)
- _****_Класс PrintMixin_****_ при создании объекта класса Product и его наследников печатает в консоль информацию о том, от какого класса и с какими параметрами был создан объект:
Например:
```
Product('Продукт1', 'Описание продукта', 1200, 10)
```
- ****Категории (Category)****
- Абстрактный класс BaseOrder является родительским классом для класса Category
- Класс хранит характеристики категории (наименование категории, описание, список товаров)

- ***Заказ (Order)***
- Абстрактный класс BaseOrder является родительским классом для класса Order 
- Класс хранит информацию о купленном товаре, его количестве и стоимости

Реализована подгрузка данных по категориями и товарам из файла JSON в модуле utils.py: 
- Функция read_json_file производит чтение файла JSON.
- Функция create_objects_from_json создавает объекты классов из файла JSON.

Реализована функция сложения товаров из одинаковых классов продуктов (полная стоимость товаров на складе), при этом проработаны ограничения сложения объектов разных классов (выбрасывается ошибка TypeError).

Реализован метод, который добавляет продукт в категорию  таким образом, чтобы не было возможности добавить вместо продукта или его наследников любой другой объект.
